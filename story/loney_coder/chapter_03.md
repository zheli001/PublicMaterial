# 第03章 版本控制的失恋

[目录](index.md) | [上一章](chapter_02.md) | [下一章](chapter_04.md)

---

我曾经以为版本控制是把代码锁进保险箱，后来才发现它也是一面镜子。每一个 commit 都像一次小小的告白：抱歉，我又改了你的函数签名；请原谅，我把你的变量名改得不伦不类。失恋不是当下分手的那一刻，而是在 pull request 的评论区看到“建议改成 XXX”的句子。

有一次，我对一个函数投入了通宵的情感，写着注释像写日记；合并时，reviewer 一句“建议简化”让我如遭雷击。我学会了在评论下加表情，也学会了在拒绝后把代码整理成可复用的模块。失恋的滋味是苦的，但它也让你学会重构自己，抽离不健康的依赖。

版本控制教我两件事：一是历史不可删（除非强制 push，但那样会留下伤痕），二是每次回滚都有机会重写叙事。于是我学会为自己保留分支，偶尔 checkout 回老地方，看曾经的幼稚，然后笑出声来.

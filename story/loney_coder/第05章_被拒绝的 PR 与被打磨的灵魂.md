第05章 被拒绝的 PR 与被打磨的灵魂

被拒绝的 PR 就像情书被退回，理由往往是“风格不合”。我收到了无数个“Close as not suitable”的通知，像是生活在一个永远不够好的循环里。起初我愤怒，后来我冷静，最后我学会感谢每一次拒绝——它们像是老师，用苛刻的注释教我写得更好。

每一次修改都是一次小小的打磨。代码被拆分成更小的函数，注释变得诚恳而具体。我的灵魂也在这个过程中被 refactor：把宏大叙事拆成可测试的小块，每次运行都能得到预期。幽默是我最好的防护：当 PR 被关闭时，我会在私聊里写一句“感谢你的严厉”，没人回复也无妨。

被拒绝不是终点，而是进入下一轮版本迭代的开始。你会发现，经过若干次的打磨，那些曾让你心碎的拒绝，成了你代码历史里最有价值的 commit.
